学习并查集（Disjoint Set Union，简称 DSU）时，掌握了你刚才问的“路径压缩”只是成功了一半。要真正拿它去解题，你需要一套完整的“思维蓝图”。

并查集的核心思想是：**维护一个森林，通过根节点来代表一个集合。**

---

### 一、 三大核心操作

除了你已经看到的 `find` 和 `union`，解题时通常需要这三个完整模块：

1. **初始化 (Init)**：
    
    每个点的父节点初始化为自己，表示起初每个元素都是独立集合。
    
    Python
    
    ```
    fa = [i for i in range(n + 1)]
    ```
    
2. **查询 (Find) + 路径压缩**：
    
    找到根节点，并把沿途所有节点的父节点直接指向根节点（即你之前问的那行代码）。
    
3. **合并 (Union)**：
    
    找到两个元素的根节点 `rootX` 和 `rootY`，如果不同，则 `fa[rootX] = rootY`。
    

---

### 二、 进阶优化：按秩合并 (Union by Rank)

如果只做路径压缩，在极端情况下（虽然概率极低）树还是可能不平衡。

**按秩合并**的思想是：每次合并时，让“矮树”嫁接到“高树”下面，这样树的高度增长最慢。

> **小技巧**：在算法竞赛中，通常只写“路径压缩”就足够应对 99% 的题目了。只有在不能使用路径压缩（如可持久化并查集）时，按秩合并才是必须的。

---

### 三、 常见的解题套路（重点）

并查集不只是用来判断“连不连通”，它的变体非常多：

#### 1. 统计连通分量的个数

- **思路**：处理完所有 `union` 后，遍历 `fa` 数组，计算有多少个 `fa[i] == i` 的节点。或者是用len(set(fa))利用集合元素唯一性
    
- **场景**：计算岛屿数量、省份数量。
    

#### 2. 带权并查集 (Weighted DSU)

- **思路**：在 `fa` 数组之外，维护一个 `value` 数组，记录节点到父节点的某种关系（如距离、倍数、胜负关系）。在 `find` 回溯时，不仅更新 `fa`，还要**累加/更新权值**。
    
- **场景**：食物链问题（A吃B，B吃C）、货币汇率转换。
    

#### 3. 维护集合的大小 (Size)

- **思路**：维护一个 `sz` 数组，只有根节点的 `sz[root]` 是有效的，表示该集合有多少个元素。合并时：`sz[rootY] += sz[rootX]`。
    
- **场景**：查找最大连通块的大小。
    

---

### 四、 避坑指南（需要注意的东西）

1. **初始化范围**：
    
    根节点是从 $0$ 开始还是从 $1$ 开始？数组开大一点（`n + 5`）总没错。
    
2. **Union 里的陷阱**：
    
    **千万不要直接 `fa[u] = v`！** 一定要先 `rootU = find(u)`, `rootV = find(v)`，然后 `fa[rootU] = rootV`。合并的是“代表元”，而不是节点本身。
    
3. **非连通图遍历**：
    
    如果你想看某个点属于哪个集合，一定要调用 `find(i)` 而不是直接读取 `fa[i]`。因为路径压缩可能还没发生，`fa[i]` 存的可能只是它临时的父节点。
    
4. **离线思维**：
    
    并查集擅长“加边”，不擅长“删边”。如果题目问删边后的连通性，可以尝试**逆向思考**：把删边看作从后往前加边。
    

---

### 五、 什么时候该想并查集？

当你看到以下关键词时，并查集应该是你的第一反应：

- “连通性”、“朋友的朋友是朋友”、“合并集合”。
    
- 最小生成树算法（Kruskal）。
    
- 判断无向图中是否存在环。
    

你目前练习的题目里，有没有遇到那种需要你在合并时顺便计算点什么（比如算集合人数）的题目？
