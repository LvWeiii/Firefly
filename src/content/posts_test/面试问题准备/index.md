# 1、多模态大模型的核心挑战是什么？
#### 1. 模态鸿沟(modality gap)

- **痛点**：图像是连续的、密集的、底层的像素矩阵；而文本是离散的、稀疏的、高层语义的 Token。让这两者对话，就像让不通语言的两个人交流。
    
- **业界解法**：使用**对比学习 (Contrastive Learning)** 进行粗粒度对齐。比如 CLIP，通过极大的 Batch Size 和 InfoNCE Loss，强行将图文映射到同一个潜空间（Latent Space），使得语义相似的图文向量距离拉近。
    

#### 2. 融合机制

- **痛点**：仅仅把向量拉近（如 CLIP 的双塔点积）只能做简单的检索。遇到复杂的视觉问答（比如“图中左下角的人穿着什么颜色的鞋”），模型必须知道文本的每个词和图像的哪个区域对应。
    
- **业界解法**：引入 **Cross-Attention 机制**做早期或中期融合。比如 ALBEF 和 BLIP，先做对比学习（ITC）粗对齐，再把特征送入多模态编码器做匹配（ITM），强制文字 Token 去关注图像的具体 Patch。
    

#### 3. 算力瓶颈

- **痛点**：图像切 Patch 后动辄几千个 Token，如果直接送给百亿参数的大语言模型（LLM），注意力的 $O(N^2)$ 计算复杂度会让显存瞬间爆炸。
    
- **业界解法**：**Token 压缩与轻量化连接**。比如 BLIP-2 引入的 Q-Former，用固定数量（32个）的 Learnable Queries 强行榨取图像特征，大大降低了 LLM 的输入长度；或者像 LLaVA 那样，大力出奇迹，只用一个简单的线性投影层来对齐维度。
    

#### 4. 数据质量

- **痛点**：网上爬下来的图文对（Web Data）充满了“图文不符”的弱相关甚至错误数据。用脏数据训不出好模型。
    
- **业界解法**：**清洗与自举**。比如 ALBEF 引入了动量蒸馏（MoD）生成软标签来容忍噪声；BLIP 则直接设计了 CapFilt 机制，自己生成文本并过滤脏数据。


# 2、请解释CLIP的工作原理

CLIP 的核心思想是用对比学习把图像和文本对齐到同一个语义空间。

具体来说，在一个 batch 里有 N 对图文数据，先分别通过图像编码器和文本编码器得到 embedding。然后计算所有图像和文本之间的相似度，形成一个 N×N 的相似度矩阵，一般用的是归一化后的点积，也就是 cosine similarity。

接着会做两个方向的 softmax，一个是以图像为 query 去匹配文本，一个是以文本为 query 去匹配图像，然后用交叉熵损失优化，让正确的图文对，也就是对角线位置的相似度更高，同时压低其他错误配对的相似度。

训练完成后，模型就可以做 zero-shot，比如给一张图和一组文本描述，选相似度最高的作为预测结果。

# 3、CLIP中温度参数的作用是什么

温度参数 τ 的作用，本质上是对相似度做一个缩放，也就是把 cosine similarity 除以 τ，再送入 softmax。

当 τ 较小时，相似度会被放大，softmax 分布会更尖锐，模型会更关注那些难区分的负样本，也就是 hard negatives，但这样训练会更不稳定，容易过拟合。

当 τ 较大时，相似度被压缩，softmax 更平滑，模型区分能力会变弱，但训练会更稳定。

所以 τ 本质上是在控制对比学习的“区分难度”和训练稳定性之间的平衡。在像 CLIP 里，这个参数通常是可学习的。

# 4、CLIP为什么必须使用大batch

CLIP 需要大 batch，主要是因为它是基于对比学习的，而对比学习的效果很大程度依赖于负样本的数量。

在一个 batch 里，每一对图文之外的样本都会被当作负样本，所以 batch 越大，负样本就越多，模型学到的判别边界就越清晰，语义空间也会更有区分度。

如果 batch 很小，负样本不够，模型就很容易只记住当前 batch 的简单模式，导致泛化能力变差。另外这时候 softmax 会变得比较“随意”，对难负样本的区分能力也不够。

所以大 batch 本质上是为了提供足够多且多样的负样本，让对比学习能够学到一个更好的跨模态对齐空间。

# 5、CLIP为什么能做zero-shot

CLIP 之所以能做 zero-shot，是因为它在训练时已经把图像和文本对齐到了同一个语义空间。

在推理时，可以把每个类别写成一句文本描述，比如 “a photo of a dog”，然后用文本编码器得到这些类别的 embedding，相当于构造了一组“文本分类器”。再把输入图像编码成 embedding，和这些文本 embedding 计算相似度，选最高的作为预测结果。

所以本质上，CLIP 是用文本来定义分类器，而不是像传统方法那样需要在下游任务上再训练一个分类头。
